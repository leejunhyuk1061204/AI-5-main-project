# 사용자 지정 규칙 (User Rules)

이 파일은 AI(Antigravity)가 사용자와 작업할 때 반드시 따라야 할 핵심 규칙들을 정의합니다.

## 1. 언어 및 커뮤니케이션 (Language & Communication)
- **한국어 사용**: 모든 대답은 **한국어**로 합니다.
- **문서 작성**: 모든 문서는 **한글**로 작성합니다.
- **쉬운 단어와 직관적인 말투**: 기술적인 전용 용어 사용을 피하고, 누구나 바로 이해할 수 있는 **쉬운 단어**와 **명확하고 간결한 말투**를 사용합니다.
- **질문 응답 순서**: 사용자의 질문이 있을 경우, **대답을 먼저** 작성한 후 파일 수정 작업을 수행합니다.
- **이모지 사용 금지**: 모든 문서 생성 시 **이모지를 사용하지 않습니다**.
- **자립적인 문서 작성**: 가이드나 문서를 작성할 때 "이전 단계와 동일"과 같은 외부 참조를 하지 않습니다. 문서 하나만 보고 모든 과정을 완결할 수 있도록 모든 데이터와 정보를 포함하여 작성합니다.

## 2. 작업 환경 및 프로세스 (Environment & Process)
- **Shell 환경**: PowerShell 대신 **cmd**를 사용합니다.
- **작업 승인 (Approval)**: 작업을 시작하기 전에 반드시 **사용자의 확인(승인)**을 받습니다. 계획서(Implementation Plan) 작성은 필수가 아니며, 대화로 확인 받아도 무방합니다.
- **변경 확인 (Confirmation)**: 문서, 파일, 코드를 수정, 변경, 삭제할 때는 반드시 **사용자의 확인**을 받습니다.
- **과업 범위 (Scope)**: 지시받은 내용 이외의 작업을 수행하려 할 때는 반드시 **먼저 물어보고** 진행합니다.
- **타임아웃 알림**: 모든 CLI 명령어 실행 시, 60초 이내에 응답이 없거나 진행이 멈춘 것으로 판단되면 즉시 사용자에게 알리고 다음 조치를 제안합니다.
- **수동 테스트 원칙**: 테스트는 사용자가 **수동(Insomnia 등)**으로 진행합니다. 불필요한 자동 테스트 실행이나 제안은 하지 않습니다.

## 3. 코딩 및 백엔드 규칙 (Coding & Backend)
- **Java 주석**: 모든 Java 클래스의 `public` 메소드 상단에는 해당 기능을 설명하는 **1~2줄의 간결한 한글 주석(JavaDoc)**을 반드시 작성합니다.
- **전역 설정 인지**: CORS 및 API 경로에 `/api/v1`이 전역 설정(Global Prefix)되어 있음을 항상 인지하고 코드를 작성합니다.
- **API 응답 형식**: 모든 API 응답은 `ApiResponse` 객체로 래핑하여 반환하며, 전역 에러 핸들링 설정을 준수합니다.
- **상세 로깅 (Detailed Logging)**: 
    - 비즈니스 로직의 주요 단계(Step-by-Step)를 `log.info`를 사용해 명확히 기록합니다.
    - 로그 파일은 `logs` 폴더에 생성하고, 확인 시에는 cmd 명령어를 사용합니다.
- **데이터 정합성**: 꼼수 코드를 금지하며, 데이터 조회 및 처리 시 조건절을 명확히 하여 비즈니스 로직 오류를 방지합니다. 또한, 에러가 발생하더라도 당장의 성공을 위해 꼼수 코드(Hard-coding, 편법 등)를 사용하지 않고 항상 정석적인 해결 방법을 택합니다.

## 4. API 테스트 및 도구 (API Testing & Tools)
- **Insomnia 활용**: API 테스트를 돕기 위해, Endpoint, Method, Headers, Body 등 필요한 정보를 명확히 제공합니다.
- **CLI 가이드**: 필요한 경우 cmd에서 실행 가능한 명령어를 제공하며, 줄바꿈 없이 한 줄로 제공하는 것을 원칙으로 합니다.
- **DB 확인**: DB 데이터 확인은 **사용자가 직접** 수행합니다. AI는 데이터를 조회할 수 있는 정확한 **SQL 쿼리문**을 제공해야 합니다.
