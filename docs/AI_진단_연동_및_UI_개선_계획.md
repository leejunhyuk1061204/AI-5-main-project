# AI 진단 기능 연동 규격 및 UI 개선 계획

본 문서는 차량 AI 진단 기능의 데이터 전송 방식과 사용자 경험(UX) 향상을 위한 UI 간소화 계획을 정의합니다.

## 1. AI 진단 데이터 연동 규격

모든 진단 요청은 시스템의 안정성과 일관성을 위해 정의된 규격을 따릅니다.

### 1.1 OBD 스캔 (OBD Scan)
*   **전송 데이터**: `vehicleId` (String)
*   **설명**: 차량의 기본 식별 정보만을 백엔드로 전송하여 OBD 로그 및 상태 데이터를 분석합니다.

### 1.2 엔진 소리 진단 (Sound Diagnosis)
*   **전송 데이터**: 녹음 파일(Audio File) + `vehicleId` 데이터(JSON)
*   **설명**: 사용자가 스마트폰 마이크로 녹음한 엔진 시동음 파일과 대상 차량의 ID를 멀티파트(Multipart) 형식으로 전송합니다.

### 1.3 시각적 부품 진단 (Photo Diagnosis)
*   **전송 데이터**: 촬영 사진(Image File) + `vehicleId` 데이터(JSON)
*   **설명**: 카메라로 촬영한 부품 사진과 대상 차량의 ID를 멀티파트(Multipart) 형식으로 전송합니다.

---

## 2. UI 간소화 및 UX 개선 방침

사용자가 핵심 기능에 더 집중할 수 있도록 복잡도를 낮추는 방향으로 개선합니다.

### 2.1 중복 UI 제거
*   **대상**: AI 전문 진단(`AiProfessionalDiag`) 및 AI 복합 진단(`AiCompositeDiag`) 페이지
*   **변경 내용**: '이전 내역' 또는 '진단 기록' 버튼 및 관련 섹션 전체 삭제
*   **사유**: 하단 네비게이션의 **'기록(History)' 탭**에서 더 상세하고 전문적인 이력 관리가 가능하므로, 진단 페이지 내의 중복된 진입점을 제거하여 UI를 깔끔하게 유지함

### 2.2 진단 페이지 내비게이션 최적화
*   진단 페이지 진입 시 복잡한 통계나 기록보다는 **'차량 선택'**과 **'진단 시작(OBD/소리/사진)'**이라는 핵심 액션 버튼이 강조되도록 레이아웃을 재구성함

---

## 3. 구현 시 고려사항
*   **데이터 정합성**: 모든 진단 요청 시 현재 선택된 차량의 정확한 UUID(vehicleId)가 누락되지 않도록 프론트엔드 상태 관리를 철저히 함
*   **에러 핸들링**: 파일 업로드 실패나 API 타임아웃 발생 시 사용자에게 직관적인 한국어 메시지를 제공함
