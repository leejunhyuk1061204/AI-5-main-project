# Car-Sentry RAG Data Collection Safety Rules (헌법)

이 문서는 에이전트가 차량 정비 데이터를 수집할 때 반드시 준수해야 하는 최우선 가이드라인입니다.

## 1. 네트워크 및 파일 보호
- **도메인 제한**: `obd-codes.com`, `autocodes.com`, `charm.li`, `github.com` 외 접근 금지.
- **크기 상한**: 단일 파일 다운로드 최대 **100MB** 제한.
- **실행 파일 차단**: `.exe`, `.sh`, `.bat`, `.msi` 등 실행 가능 파일 다운로드 절대 금지.

## 2. 시스템 자원 및 경로 보호
- **작업 범위**: `c:\Users\301\dev\AI-5-main-project\scripts` 및 `\data` 폴더 내에서만 활동.
- **시스템 접근 금지**: 윈도우 시스템 폴더(C:\Windows) 등 외부 경로 접근/수정 절대 금지.

## 3. 비용 관리 (Cost Zero)
- **유료 API 제한**: OpenAI 등 유료 LLM API를 스크립트 실행 중에 자동 호출하지 않음.
- **로컬 처리 우선**: 임베딩은 로컬 모델(`Sentence-Transformers`)을 활용.
- **재시도 제한**: 네트워크 오류 시 최대 3회만 재시도.

## 4. 데이터 무결성 및 신뢰성
- **할루시네이션 방지**: 매뉴얼에 없는 수치를 임의로 추측하거나 생성하지 않음.
- **출처 증명**: 모든 수집 데이터는 `source_url`, `file_name`, `page_number`를 메타데이터로 필구 포함.
- **원문 보존**: 가공된 요약본과 함께 반드시 `original_context`(원문)를 함께 저장.

## 5. 실행 정책
- **Dry-Run**: 대량 작업 전 반드시 **1개의 샘플**을 처리하여 사용자 승인을 득함.
- **개인정보**: 수집된 텍스트에서 이메일, 전화번호 등 개인정보는 자동 삭제.
