# 프론트엔드-백엔드 연결 상태 분석 보고서

현재 프로젝트의 프론트엔드와 백엔드 간 API 연결 상태를 분석한 결과입니다.

## 1. 연결된 기능 (Connected Features)
현재 프론트엔드에서 API 연동이 구현되어 있는 기능들입니다.

| Backend Controller | Frontend API File | 주요 기능 |
|-------------------|-------------------|----------|
| **AuthController** | `services/auth.ts` | 회원가입, 로그인, 프로필 조회/수정/삭제 |
| **VehicleController** | `api/vehicleApi.ts` | 차량 등록(수동/OBD), 목록 조회, 상세 조회, 수정, 대표/삭제 |
| **ObdController** | `api/obdApi.ts` | OBD 데이터 배포 전송, 연결 상태 확인/해제 |

## 2. 미연결 기능 (Unconnected Features)
백엔드에는 컨트롤러가 존재하지만, 프론트엔드에서 API 호출 코드가 발견되지 않은 기능들입니다.
이 기능들은 현재 프론트엔드에서 **더미 데이터**를 사용하고 있거나 **구현되지 않았을** 가능성이 높습니다.

| Backend Controller | 주요 기능 (Endpoint) | Frontend 상태 |
|-------------------|-------------------|--------------|
| **AiController** | `POST /ai/diagnose` (AI 진단 요청)<br>`POST /ai/dtc` (DTC 코드 처리) | **미연결**: 비주얼/소리 진단 화면에서 API 호출 없음. |
| **MaintenanceController** | `POST .../maintenance` (정비 기록 등록)<br>`GET .../consumables` (소모품 상태 조회)<br>`POST .../ocr` (영수증 분석) | **미연결**: 정비 내역 관리 및 영수증 OCR 기능 연동 필요. |
| **TripController** | `GET /trips` (주행 이력 조회)<br>`POST /trips/start`, `end` (주행 시작/종료) | **미연결**: 주행 이력 그래프 및 목록, 주행 시뮬레이션 연동 필요. |
| **NotificationController** | `GET /notifications` (알림 목록)<br>`PATCH .../read` (읽음 처리) | **미연결**: 알림 센터 기능 연동 필요. |
| **CloudAuthController** | 클라우드 연동 인증 | **미연결** |
| **AdminController** | 관리자 기능 | **미연결** (사용자 앱에는 불필요할 수 있음) |
| **MediaController** | 미디어 파일 업로드/관리 | **미연결** (AI 진단 시 파일 업로드에 필요할 수 있음) |
| **MasterController** | 마스터 데이터 관리 | **미연결** |

## 3. 상세 분석 및 조치 필요 사항

### 1) AI 진단 (AiController)
- **현상**: 비주얼 진단(`VisualDiagnosis.tsx`) 및 소리 진단(`SoundDiagnosis.tsx`) 화면은 존재하나, 실제 파일 업로드 및 분석 요청 API가 연결되지 않았습니다.
- **필요 조치**: 프론트엔드에 `aiApi.ts`를 생성하고 `AiController`의 엔드포인트를 호출하도록 구현해야 합니다.

### 2) 정비 관리 (MaintenanceController)
- **현상**: 소모품 관리(`ConsumableManage.tsx`) 및 정비 내역 화면에서 백엔드 데이터를 불러오지 않고 있습니다.
- **필요 조치**: `maintenanceApi.ts` 생성 및 소모품 상태/정비 이력 CRUD/OCR 분석 연동이 시급합니다.

### 3) 주행 이력 (TripController)
- **현상**: 주행 이력 그래프(`DrivingHis.tsx`)가 더미 데이터로 동작하고 있을 가능성이 큽니다.
- **필요 조치**: `tripApi.ts` 생성 및 주행 데이터 시각화 연동이 필요합니다.

## 4. 권장 작업 순서
1. **AI 진단 연결**: 프로젝트의 핵심 기능인 AI 진단(이미지/소리) 연동.
2. **주행 이력 연결**: 주행 습관 및 안전 점수 시각화 연동.
3. **정비/소모품 관리 연결**: 영수증 OCR 및 소모품 주기 관리 연동.
