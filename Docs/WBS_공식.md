# 프로젝트 WBS (v1.0)

**프로젝트 명**: 차량 예지 정비 시스템 (AI 기반 예지 정비 플랫폼)
**기간**: 7주 (개발 5주 + 통합 1주 + 최종 마무리 1주)
**목표**: OBD-II 데이터와 AI 멀티모달 진단을 결합한 상용화 수준의 서비스 구축

---

## 📅 주차별 주요 마일스톤

| 주차 | 단계 | 핵심 목표 |
|:---:|:---:|:---|
| **1주차** | **기반 구축** | 개발 환경 셋업, DB 스키마 확정, AI 기초 모델 선정 및 데이터 준비 |
| **2주차** | **핵심 기능 (1)** | OBD 데이터 수집 엔진(BE), 모바일 대시보드(FE) 기초, AI 전이 학습 착수 |
| **3주차** | **핵심 기능 (2)** | 차량 상세 진단 API, 실시간 모니터링 고도화, AI 모델 1차 학습 완료 |
| **4주차** | **서비스 통합** | AI 멀티모달 융합(Audio/Image/Data), 부가 기능(알림, 차계부) 구현 |
| **5주차** | **시스템 고도화** | 상용화 수준의 예외 처리 및 성능 최적화, 데모 시나리오 완성 |
| **6주차** | **품질 검증** | 통합 테스트, 버그 수정 및 QA, 인프라 배포 최적화 |
| **7주차** | **최종 발표** | 최종 성과물 정리, 시연 영상 및 발표 자료 작성 |

## 👥 팀별 업무 분담 (개발 5주)

### 1. Backend (BE)
- **1주차**: Spring Boot 초기 설정, TimescaleDB/PostgreSQL 설계 및 구축.
- **2~3주차**: OBD 텔레메트리 스트리밍 API, 사용자/차량 관리, 블루투스 인증 관리.
- **4주차**: 공공 데이터(리콜/제원) 연동, AI 서비스(FastAPI) 통신 인터페이스 구축.
- **5주차**: 보안 강화(민감정보 암호화), 대량 데이터 배치 분석 로직, 데모 데이터 주입기.

### 2. Frontend (FE)
- **1주차**: 모바일 앱 아키텍처 수립 및 디자인 시스템 적용.
- **2~3주차**: 블루투스 페어링 모듈, 실시간 차량 모니터링 대시보드 UI.
- **4주차**: AI 진단 요청(사진 촬영/녹음) UI, 인터렉티브 진단 리포트 화면.
- **5주차**: 푸시 알림 연동, 사용자 경험(UX) 최적화 및 최종 폴리싱.

### 3. AI Team (AI)
- **1주차**: Vision/Audio/TimeSeries 전이 학습 모델(SOTA) 선정 및 데이터 전처리.
- **2~3주차**: 
  - **Vision**: 외관 손상 및 마모 분석 모델 학습.
  - **Audio**: 엔진 이음 분류 모델 학습.
  - **TimeSeries**: 배터리/연료 이상 탐지 모델 학습.
- **4주차**: 멀티모달 융합 로직 구현, GPT-4o 기반 자연어 리포트 생성 최적화.
- **5주차**: API 추론 속도 최적화, 모델 정확도 검토 및 최종 배포.
